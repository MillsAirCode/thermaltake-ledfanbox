# Contributing to Thermaltake LEDFanBox Controller

Thank you for your interest in contributing to this project! Your help is appreciated.

## How to Contribute

### 1. Fork the Repository

Fork the repository on GitHub to your own account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/thermaltake-ledfanbox.git
cd thermaltake-ledfanbox
```

### 3. Create a Branch

```bash
git checkout -b feature/AmazingFeature
```

### 4. Make Your Changes

- Write clean, well-commented code
- Follow the existing coding style
- Add tests for new features
- Update documentation as needed

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add some AmazingFeature"
```

Use conventional commit messages:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for code formatting changes
- `refactor:` for code refactoring
- `test:` for adding or updating tests
- `chore:` for maintenance tasks

### 6. Push to Your Fork

```bash
git push origin feature/AmazingFeature
```

### 7. Open a Pull Request

1. Go to the original repository
2. Click "New Pull Request"
3. Select your branch
4. Fill in the PR description
5. Submit the PR

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on what's best for the community
- Show empathy towards other community members
- Accept constructive criticism

## Development Guidelines

### Code Style

- Follow PEP 8 style guide
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Testing

- Test your changes thoroughly
- Add unit tests for new functionality
- Test on multiple platforms if possible
- Verify the device still works correctly

### Documentation

- Update README.md if adding features
- Add docstrings to new functions
- Update CHANGELOG.md with changes
- Add examples for new features

## Reporting Bugs

When reporting a bug, please include:

1. **A clear and concise title**
2. **A detailed description** of the bug
3. **Steps to reproduce** the issue
4. **Expected behavior** vs **actual behavior**
5. **Environment details** (OS, Python version, etc.)
6. **Relevant logs** or error messages
7. **Screenshots** (if applicable)

## Feature Requests

When suggesting a feature, please include:

1. **Clear description** of the requested feature
2. **Use cases** for the feature
3. **How it would be implemented** (optional)
4. **Priority** (low/medium/high)

## Pull Request Guidelines

### Before Submitting

- Ensure all tests pass
- Run the linter if available
- Update documentation
- Add or update tests
- Check for code style issues

### PR Description

Include:

- **Summary** of changes
- **Related issues** (if any)
- **Screenshots** (if applicable)
- **Testing instructions** (if needed)

### Code Review

- Address review comments
- Make requested changes
- Re-submit for review
- Keep communication open

## Getting Help

If you need help:

- Check existing issues and documentation
- Open a new issue with your question
- Ask in the discussion section
- Contact the maintainer directly
